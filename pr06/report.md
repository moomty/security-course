Строка: `12:34:56.789012 IP 127.0.0.1.44321 > 127.0.0.1.8080: Flags [S], seq 123, length 0`

| Поле | Значение | Что означает |
|------|----------|--------------|
| 12:34:56.789012 | 12:34:56.789012 | Время захвата пакета с микросекундами |
| IP | IP | Протокол сетевого уровня (Internet Protocol) |
| 127.0.0.1.44321 | 127.0.0.1.44321 | IP-адрес отправителя (127.0.0.1) и порт отправителя (44321) |
| Flags [S] | [S] | TCP-флаг SYN — начало установки соединения |
| length 0 | 0 | Длина полезной нагрузки (данных нет, только заголовок) |

## 2. Перехваченные данные HTTP

![Скриншот](screens/7.png)
Чем tshark отличается от tcpdump? Когда использовать каждый?
tcpdump — это простой сниффер, который работает только с BPF-фильтрами (порты, IP-адреса) и показывает сырые заголовки пакетов. Он не распознаёт протоколы выше TCP/IP.
tshark — это консольная версия Wireshark. Он понимает структуру протоколов (HTTP, DNS, TLS) и поддерживает display-фильтры

| Что наблюдаем | HTTP | HTTPS |

## 6. Base64

![Скриншот](screens/15.png)

Почему Base64 не является шифрованием: это кодирование, а не шифрование. У него нет ключа, оно обратимо без каких-либо вычислений. Любой может декодировать строку обратно командой `base64 -d`.

## 7. tshark vs tcpdump

| Инструмент | Когда использовать |
|------------|-------------------|
| tcpdump | Быстрый захват трафика в консоли, на серверах без GUI |
| tshark | Автоматизация, анализ pcap-файлов в консоли, серверы без GUI |
| Wireshark | Детальный визуальный анализ, Follow TCP Stream, графический интерфейс |

## 8. Каналы утечки через сеть

Какие меры защиты закрывают сетевой канал утечки: шифрование трафика (HTTPS, SSH, VPN), использование защищённых версий протоколов (SFTP вместо FTP, SSH вместо Telnet), настройка Dynamic ARP Inspection на коммутаторах, сегментация сети (VLAN), использование DNS over HTTPS/TLS.

## Контрольные вопросы

78. Что такое promiscuous mode и зачем он нужен снифферу?
Promiscuous mode — режим сетевого интерфейса, при котором он принимает все пакеты в сети, а не только адресованные его MAC-адресу. Нужен снифферу для перехвата трафика других устройств.

79. Чем отличается capture filter от display filter в Wireshark?
Capture filter применяется до захвата и определяет, какие пакеты сохранять. Display filter применяется после захвата и только скрывает пакеты из интерфейса, не удаляя их из файла.

80. Что такое Follow TCP Stream и для чего используется?
Это функция Wireshark, которая собирает все пакеты одного TCP-соединения и показывает их как единый диалог. Используется для восстановления содержимого сессии.

81. Почему Base64 в HTTP Basic Auth не является защитой? Что нужно использовать вместо?
Base64 — это кодирование, а не шифрование. Оно легко декодируется обратно. Вместо него нужно использовать HTTPS (TLS).

82. Какие данные утекают даже при использовании HTTPS? Как это исправить?
Утекают: IP-адреса, порты, объём трафика, SNI (имя домена), DNS-запросы. Исправление: DNS over HTTPS/TLS для DNS, ESNI/ECH для скрытия SNI.

83. Сисадмин говорит: «у нас свитч, а не хаб — значит сниффинг невозможен». Он прав?
Нет. Сниффинг возможен через ARP-спуфинг, порт-зеркалирование (SPAN), MAC-флудинг.

84. Напишите фильтр tcpdump который перехватывает только DNS-запросы (UDP порт 53) от конкретного IP 192.168.1.50.
sudo tcpdump -i eth0 -n port 53 and host 192.168.1.50
