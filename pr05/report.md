## 2. Шифрование

Зашифрованный файл (первые 5 строк):

-----BEGIN PGP MESSAGE-----
[ЗАШИФРОВАННЫЕ ДАННЫЕ]
...
-----END PGP MESSAGE-----



Можно ли прочитать исходные данные из зашифрованного файла:
Нет. Зашифрованный файл содержит только случайный набор символов. Исходные данные можно восстановить только с помощью приватного ключа и пароля.

Что произойдёт если ввести неверный пароль при расшифровке:
Расшифровка завершится ошибкой `gpg: decryption failed: Bad passphrase`. Без правильного пароля приватный ключ не может быть использован.

Почему нельзя расшифровать файл зашифрованный для другого пользователя:
Файл зашифрован на публичном ключе получателя. Расшифровать его может только владелец соответствующего приватного ключа. Если у меня нет приватного ключа соседа — я не могу расшифровать файл.

## 3. Цифровая подпись

Вывод gpg --verify (Good signature):

![Скриншот Good signature](screens/5.png)

Команда проверяет подпись файла.
Почему подпись перестала быть действительной после изменения файла:
Подпись создаётся на основе хеша файла. При изменении файла меняется его хеш, и подпись перестаёт соответствовать содержимому. Это доказывает, что файл был изменён после подписания.

Что именно проверяет gpg --verify:
Команда проверяет, что подпись соответствует файлу и что она создана тем пользователем, чей публичный ключ используется. Она подтверждает целостность файла и авторство.

## 5. rm vs shred

| Операция | Что происходит с данными на диске | Можно восстановить? |
| :--- | :--- | :--- |
| rm | Удаляется только запись в метаданных (inode). Данные остаются на диске до перезаписи. | Да, пока блоки не перезаписаны |
| shred | Содержимое блоков перезаписывается случайными данными (3+ проходов), затем нулями, после чего файл удаляется. | Нет |

Что именно делает rm с файлом:
rm удаляет ссылку на файл из файловой системы и освобождает inode. Сами данные остаются на диске до тех пор, пока операционная система не перезапишет эти блоки новыми данными.

Чем shred отличается от rm на уровне работы с блоками диска:
shred перезаписывает блоки файла перед удалением — данные физически уничтожаются. rm только удаляет указатель на файл, оставляя данные доступными для восстановления.

Сценарий когда нужно затирать свободное место:
При продаже или передаче диска другому человеку, чтобы исключить восстановление ранее удалённых файлов. Также при уничтожении носителей с секретными данными по требованиям безопасности.

## Контрольные вопросы

68. В чём разница между шифрованием и подписью? Можно ли одновременно зашифровать и подписать файл?
Шифрование защищает содержимое от посторонних. Подпись защищает авторство и целостность. Можно одновременно зашифровать и подписать файл командой `gpg --sign --encrypt --recipient EMAIL file`.

69. Почему публичный ключ можно раздавать всем, а приватный нельзя никому?
Публичный ключ предназначен для шифрования и проверки подписи. Приватный ключ используется для расшифровки и создания подписи. Если приватный ключ попадёт к злоумышленнику, он сможет расшифровать данные и подписываться от вашего имени.

70. Что произойдёт с зашифрованными данными если потерять приватный ключ?
Данные станут нерасшифруемыми. Без приватного ключа невозможно восстановить исходный текст. Именно поэтому важны резервные копии ключей.

71. Можно ли подделать Verified коммит на GitHub не имея приватного GPG-ключа владельца?
Нет. Подпись коммита создаётся с помощью приватного ключа. Без него невозможно создать подпись, которую GitHub подтвердит как Verified.

72. Почему shred плохо работает на SSD? Что использовать вместо него?
На SSD используется wear leveling — данные могут быть записаны в другие физические блоки, и оригинальные блоки остаются неизменными. Вместо shred рекомендуется использовать шифрование всего диска или команду `blkdiscard` для SSD.

73. Коллега говорит: «я отформатировал диск перед продажей — всё безопасно». Он прав?
Нет. Быстрое форматирование удаляет только таблицу разделов. Данные остаются на диске и могут быть восстановлены. Для безопасного уничтожения нужно использовать низкоуровневое форматирование или физическое уничтожение носителя.

74. Чем отличается detach-sign (--detach-sign) от встроенной подписи (--sign)?
detach-sign создаёт отдельный файл подписи (.asc), который передаётся вместе с файлом. sign встраивает подпись в сам файл (создаёт .gpg файл). Отдельная подпись удобна для проверки без изменения исходного файла.
