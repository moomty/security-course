# ПР №2. Нормативная база и анализ CVE

## 1. Нормативные документы

### ГОСТ Р 50739-95
- Область применения: Устанавливает требования к защищенности средств вычислительной техники от несанкционированного доступа к информации.
- Количество классов защищённости: В документе определено 7 классов защищенности (от 1-го  самого высокого, до 7-го  базового).
- Что регулирует управление доступом: Требования к идентификации и аутентификации пользователей, разграничению доступа к ресурсам и регистрации событий безопасности.

### Приказ ФСТЭК №17
- Кому адресован: Федеральным органам исполнительной власти, государственным органам и организациям, создающим государственные информационные системы (ГИС).
- Группа мер УПД (Управление правами доступа) включает:
  - Управление учетными записями пользователей.
  - Назначение прав доступа на основе ролевой модели.
  - Контроль действий пользователей с повышенными привилегиями.
  - Ограничение несанкционированного доступа к ресурсам системы.

### Сравнение ГИС и ИСПДн
| Характеристика | ГИС | ИСПДн |
| :--- | :--- | :--- |
| Регулирующий документ | Приказ ФСТЭК №17 | Приказ ФСТЭК №21 |
| Объект регулирования | Государственные информационные системы | Системы, обрабатывающие персональные данные |
| Уровни защищенности | Не классифицируются по уровням | 4 уровня (УЗ-1 … УЗ-4) |

## 2. Анализ уязвимости
**CVE-идентификатор:** CVE-2024-6387  
**Название:** regreSSHion  
**CVSS Score:** 8.1 (Высокий уровень)  
**Затронутый продукт:** OpenSSH (версии < 9.8)

### Описание
Уязвимость в серверной части OpenSSH (sshd), связанная с состоянием гонки (race condition) в процедуре аутентификации. При успешной эксплуатации злоумышленник может выполнить произвольный код с правами root на целевой системе.

### Механизм атаки
Атака использует race condition между двумя процессами аутентификации, конкурирующими за одни и те же ресурсы. Злоумышленник отправляет серию специально сформированных запросов, чтобы воспользоваться временным окном, в котором система находится в некорректном состоянии. В результате возможно выполнение кода с максимальными привилегиями.

**CVSS вектор:** `AV:N/AC:H/PR:N/UI:N/S:U/C:H/I:H/A:H`

| Компонент | Значение | Что означает |
| :--- | :--- | :--- |
| Attack Vector | Network (AV:N) | Атака возможна удаленно через сеть |
| Attack Complexity | High (AC:H) | Требует специфических условий (сложный тайминг) |
| Privileges Required | None (PR:N) | Аутентификация не требуется |
| User Interaction | None (UI:N) | Действия пользователя не нужны |
| Impact (C/I/A) | High/High/High | Полный компромисс системы |

### Применимые меры защиты по Приказу ФСТЭК №17
| Группа мер | Мера | Как помогает |
| :--- | :--- | :--- |
| ОПС | Своевременное обновление OpenSSH до версии 9.8 или выше | Устраняет уязвимость на уровне кода, патч от разработчика |
| ЗИС | Ограничение доступа к SSH-серверу по IP-адресам (firewall) | Снижает риск эксплуатации из недоверенных сетей |
| ИАФ | Использование аутентификации по SSH-ключам вместо паролей | Сложность эксплуатации увеличивается, так как ключи более надежный механизм |
